# Инструкция пользователя: мониторинг участия в ОПРЧ

Файлы модуля Excel VBA:
- `OPRCH_Monitor.bas` — версия в `UTF-8` (для Git/редакторов).
- `OPRCH_Monitor_win1251.bas` — версия в `Windows-1251` (для импорта в VBA на русской Windows).

Макрос выполняет:
- количественный мониторинг участия в ОПРЧ;
- качественный мониторинг участия в ОПРЧ;
- отдельные вкладки по каждому генератору;
- отдельные вкладки суммарной нагрузки станции для:
  - Сосногорской ТЭЦ,
  - Воркутинской ТЭЦ,
  - СЛПК.

## 1. Как запустить

1. Создайте книгу Excel в формате `.xlsm`.
2. Откройте VBA-редактор: `Alt + F11`.
3. Импортируйте модуль: `File -> Import File...` -> `OPRCH_Monitor.bas`.
4. Запустите `SetupOPRCHTemplate`.
5. Заполните листы `RawData` и `Config`.
6. На листе `Config` нажмите кнопку `Запустить мониторинг ОПРЧ` (или запустите `AnalyzeOPRCH` вручную).

## 2. Формат входных данных (`RawData`)

### Обязательные столбцы
- `Время` — дата/время Excel.
- Частота — может быть:
  - общий столбец `Частота`,
  - или отдельные столбцы, например `f Блок 1`, `f Блок 2`, `f СЛПК`.
- Мощность каждого генератора — отдельный столбец (например `ТГ-5`, `Р Блок 1`, `P ТГ-3Э`).

### Правила
- Заголовки только в строке 1.
- Данные идут без разрывов по времени в анализируемом интервале.
- Имена столбцов мощности/частоты должны совпадать с тем, что указано в `Config`.

## 3. Параметры в `Config`

Таблица с генераторами начинается со строки 2.

Колонки:
- `A Станция`
- `B Генератор`
- `C Колонка_мощности` (точное имя столбца из `RawData`)
- `D Колонка_частоты` (точное имя столбца из `RawData`)
- `E Тип_оборудования` (например: ПТУ, ГТУ, ПГУ_утилиз и т.д.)
- `F Pном, МВт`
- `G S, %`
- `H fнч, Гц`
- `I Kд`
- `J Вкл (1/0)` — участвует ли генератор в расчете
- `K Кач_вкл (1/0)` — включать ли качественную проверку
- `L t5, c` — нормативное время достижения первой ступени
- `M dP5, %Pном` — первая ступень по мощности
- `N t10, c` — нормативное время достижения второй ступени
- `O dP10, %Pном` — вторая ступень по мощности
- `P Уст_допуск, %Pном` — допуск установившегося отклонения
- `Q В сумму станции (1/0)` — включать в суммарный мониторинг станции
- `R Контр_уст (1/0)` — контроль критерия установившегося значения:
  - `1` — учитывать,
  - `0` — не учитывать (критерий отключен для строки генератора).

Глобальные параметры:
- `T2 fном, Гц` (обычно 50)
- `T3 Время начала события` (используется при ручном старте)
- `T4 Автопоиск старта (1/0)`
  - `1`: старт = предыдущая точка перед выходом частоты за `fнч`;
  - `0`: старт = ближайшая к времени `T3` точка.
- `T5 Колич. интервал, с`
- `T6 Допуск количеств., %`
- `T7 Порог включения в работу, МВт` (для суммарной нагрузки станции)

## 4. Алгоритм макроса

Для каждого включенного генератора (`J=1`):

1. Определяет старт анализа:
   - авто: предыдущая точка перед выходом частоты за `fнч`;
   - вручную: ближайшая точка к времени `T3`.
2. Количественная оценка:
   - на интервале `T5` секунд считает:
     - `Δf`, `Δfр`,
     - `Pтреб = -100/S * Pном/fном * Kд * Δfр`,
     - `Pфакт = Pтек - P0`,
   - проверяет знак и уровень `|Pфакт|` относительно `|Pтреб|` с допуском `T6`.
3. Качественная оценка (если `K=1`):
   - проверяет достижение ступеней:
     - `dP5` за `t5`,
     - `dP10` за `t10`,
  - проверяет установившееся значение с допуском `Уст_допуск` только если `R=1`.
4. Формирует отдельный лист генератора:
   - расчетные показатели;
   - таблицу временного ряда;
   - график `ΔPфакт`, `Pтреб`, `Частота`.

Для станций Сосногорская ТЭЦ, Воркутинская ТЭЦ, СЛПК:
- строит отдельный лист суммарного мониторинга по генераторам, у которых `Q=1`;
- в сумму включает генераторы с фактической мощностью выше порога `T7`.

## 5. Результаты

- Лист `Summary`:
  - свод по всем генераторам: количественный и качественный статус;
  - отдельные подпункты качества:
    - `Кач.t5`,
    - `Кач.t10`,
    - `Кач.уст`,
  - `Проваленные подпункты` (может содержать сразу несколько критериев).
- Отдельный лист на каждый генератор:
  - имя вида `Станция_Генератор`.
- Отдельные листы суммы станции:
  - `Сосногорская ТЭЦ_Сумма`,
  - `Воркутинская ТЭЦ_Сумма`,
  - `СЛПК_Сумма`.

## 6. Типовые ошибки

- `Не найдена колонка мощности ...`  
  Проверьте `Config!C` и заголовок в `RawData`.

- `Не найдена колонка частоты ...`  
  Проверьте `Config!D` и заголовок в `RawData`.

- Пустые/некорректные результаты  
  Проверьте обязательные параметры: `Pном`, `S`, `fнч`, `Kд`, а также формат времени.
