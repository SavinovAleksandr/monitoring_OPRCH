# Инструкция пользователя: мониторинг участия в ОПРЧ

Версия макроса: **1.5.0**

Файлы модуля Excel VBA:
- `OPRCH_Monitor.bas` — версия в `UTF-8` (для Git/редакторов).
- `OPRCH_Monitor_win1251.bas` — версия в `Windows-1251` (для импорта в VBA на русской Windows).

Макрос выполняет:
- количественный мониторинг (с контролем перерегулирования);
- качественный мониторинг (t5, t10, установившееся среднее по хвосту);
- оценку характера переходного процесса (Монотонный / Апериодический / Колебательный);
- оценку амплитуды события в %Pном и флаг масштаба (Слабое / Норма / Избыточное);
- учёт максимальной и минимальной мощности (`Pmax` / `Pmin`): капинг `Pтреб` располагаемым резервом, статус `Ограничен Pmax/Pmin`, предупреждение при резерве < 5 % Pном, горизонтальные уровни на графике. Для станций с общим паропроводом (в т.ч. по паропроводам `У` / `Э` ТЭЦ СЛПК) `Pmax_сум` / `Pmin_сум` = сумма по включённым генераторам;
- отдельные вкладки с данными и графиками по каждому генератору;
- суммарный мониторинг станции (по флагу `Q=1`), в т.ч. по общим паропроводам;
- валидацию RawData/Config с отдельным листом `Log`;
- цветовое оформление `Summary` с подсветкой нарушений и ограничений;
- настраиваемый профиль `Kд(t)` по типу оборудования через `Config` (колонки `AA:AI`) с 4 точками времени и выводом примененного профиля в листах генератора и графика;
- два графика на листе `<Ст>_<Ген>_Граф`: верхний в относительных единицах (`dP`), нижний в абсолютных (`P, МВт` + `f, Гц`).
- настройку состава рядов графиков через `Config` (чекбоксы в колонках `AK:AL`).

## 1. Как запустить

1. Создайте книгу Excel в формате `.xlsm`.
2. Откройте VBA-редактор: `Alt + F11`.
3. Импортируйте модуль: `File -> Import File...` → `OPRCH_Monitor.bas` (или `_win1251.bas` на русской Windows).
4. Запустите `SetupOPRCHTemplate` — появятся листы `RawData`, `Config`, `Summary` и три кнопки на листе `Config`:
   - **Запустить мониторинг ОПРЧ** → `AnalyzeOPRCH`;
   - **Очистить результаты** → `ClearOPRCHResults` (удаляет все вкладки генераторов и аггрегатов, оставляя `RawData`, `Config`, `Summary`, `Log`);
   - **Применить пресеты типов** → `ApplyPresetsToConfig` (подставляет t5/t10/dP5/dP10/Уст_допуск/fнч по типу оборудования в пустые ячейки).
5. Заполните листы `RawData` и `Config` (см. раздел 3).
6. Нажмите **Запустить мониторинг ОПРЧ**.

## 2. Формат входных данных (`RawData`)

### Обязательные столбцы
- `Время` — дата/время Excel.
- Частота — один из вариантов:
  - общий столбец `Частота`,
  - или отдельные столбцы, например `f Блок 1`, `f Блок 2`, `f СЛПК`.
- Мощность каждого генератора — отдельный столбец (например `ТГ-5`, `Р Блок 1`, `P ТГ-3Э`).

### Правила
- Заголовки — только в строке 1.
- Данные в анализируемом интервале идут без разрывов по времени.
- Имена столбцов мощности/частоты должны точно совпадать с тем, что указано в `Config`.
- Рекомендуемый шаг данных — не более 5 с (в противном случае в `Log` появится предупреждение `WARN Разрыв по времени ...`).

## 3. Параметры в `Config`

### 3.1 Таблица генераторов (со строки 2, колонки A..T)

| Кол. | Поле | Пояснение |
|------|------|-----------|
| A | Станция | |
| B | Генератор | |
| C | Колонка_мощности | точное имя столбца из `RawData` |
| D | Колонка_частоты | точное имя столбца из `RawData` |
| E | Тип_оборудования | `ПТУ_неблок`, `ПТУ_блок`, `ГТУ`, `ПГУ_утил`, `ПГУ_сбросн`, `ГПА` |
| F | Pном, МВт | |
| G | S, % | |
| H | fнч, Гц | |
| I | Вкл (1/0) | участвует ли генератор в расчёте |
| J | Кач_вкл (1/0) | включать ли качественную проверку |
| K | t5, c | нормативное время первой ступени |
| L | dP5, %Pном | первая ступень по мощности |
| M | t10, c | нормативное время второй ступени |
| N | dP10, %Pном | вторая ступень по мощности |
| O | Уст_допуск, %Pном | допуск установившегося отклонения |
| P | В сумму станции (1/0) | включать в суммарный мониторинг станции |
| Q | Контр_уст (1/0) | контролировать ли установившееся значение |
| R | Паропровод | тег общего паропровода (например `Э`, `У`) |
| S | Pmax, МВт | максимальная эксплуатационная мощность (по умолчанию `Pном`) |
| T | Pmin, МВт | технический минимум (по умолчанию `0`) |

**Pmax / Pmin.** Если в `Config` заданы `T` и `U`, макрос:
- вычисляет располагаемый резерв `Резерв '+' = max(0, Pmax − P0)` и `Резерв '-' = max(0, P0 − Pmin)`;
- ограничивает `Pтреб` резервом в нужную сторону (капинг): если исходный `Pтреб` больше резерва, принимается значение резерва;
- в `Summary` количественный статус становится `Ограничен Pmax` / `Ограничен Pmin` (вместо `Нарушение`), в колонке `Pтреб исх., МВт` сохраняется неограниченное значение, в колонке `Ограничение` — тип лимита;
- при резерве в требуемую сторону `< 5 % Pном` в `Log` записывается `WARN` и событие помечается как возможное ограничение диапазоном;
- на графике `<Ст>_<Ген>_Граф` добавляются красные пунктирные уровни `ΔPmax` и `ΔPmin` (относительно `P0`).

Если `T` и `U` пусты — Pmax принимается равным `Pном`, Pmin = 0 (полный диапазон, капинг фактически отключён).

**Пресеты типов.** Если ячейки `L..P` или `H` (fнч) оставлены пустыми/нулевыми, при запуске они заменяются на значения по умолчанию в зависимости от `E Тип_оборудования` — см. таблицу ниже (колонку `t10` можно быстро заполнить для всех строк кнопкой **Применить пресеты типов**):

| Тип | t10, c | t5 | dP5 | dP10 | Уст_допуск | fнч |
|-----|-------:|---:|----:|-----:|-----------:|----:|
| ПТУ_неблок | 420 | 15 | 5 | 10 | 1 | 0.075 |
| ПТУ_блок   | 360 | 15 | 5 | 10 | 1 | 0.075 |
| ГТУ        | 900 | 15 | 5 | 10 | 1 | 0.075 |
| ПГУ_утил   | 900 | 15 | 5 | 10 | 1 | 0.075 |
| ПГУ_сбросн | 2100| 15 | 5 | 10 | 1 | 0.075 |
| ГПА        | 120 | 15 | 5 | 10 | 1 | 0.075 |

Пресеты срабатывают только для пустых ячеек — любые заданные значения сохраняются.

| W3/X3 | Время начала события (ручной старт) | — |

| W7/X7 | Порог включения в работу, МВт | 1 |

| W10/X10 | Окно установив., с | 30 |

> В 1.4.0 блок глобальных настроек перенесён из `T/U` в `W/X`, чтобы освободить `T/U` для `Pmax/Pmin` каждого генератора. Старые книги (1.3.x) продолжают читаться — макрос сам распознаёт старое расположение и подхватывает значения из `T/U`, если `W` пустой.

- **Pre-start буфер** — сколько секунд добавить в график **до** расчётной точки старта. На расчёт не влияет, только на ширину окна графика.
- **Интервал графика** — длительность окна графика **после** расчётной точки старта.
- **Окно установившегося** — на какое окно (последние N секунд качественного интервала) усредняется установившееся значение мощности.

### 3.3 Профили `Kд(t)` (колонки `AA:AI`)

Таблица профилей (со строки 3):
- `AA` — `Тип_оборудования` (например `ПТУ_блок`, `ПТУ_неблок`, `ГТУ`, `ПГУ_утил`, `ПГУ_сбросн`, `ГПА`);
- `AB/AC` — точка `(t0, Kд0)`;
- `AD/AE` — точка `(t1, Kд1)`;
- `AF/AG` — точка `(t2, Kд2)`;
- `AH/AI` — точка `(t3, Kд3)`.

Как используется:
- `Kд(t)` считается кусочно-линейно по заданным точкам времени; минимум 2 точки должны быть заполнены, остальные можно оставить пустыми;
- профиль `AB:AI` обязателен: если для типа оборудования нет строки профиля, расчёт завершается с ошибкой;
- столбец фиксированного `Kд` в таблице генераторов удалён; источник `Kд(t)` — только профиль `AB:AI`;
- профиль применяется и в количественном расчете (`Pтреб`), и в расчете `Pтреб_ср` для установившегося режима.

Шаблон по умолчанию содержит последовательные точки `t0=0 c`, `t1=30 c`, `t2=240 c (4 мин)`, `t3` далее.
Для паросиловых блоков (`ПТУ_блок`/`ПТУ_неблок`) задан ориентир:
- `Kд1 = 0.5` на `30 c`;
- `Kд2 = 0.8` на `240 c`.

### 3.4 Выбор рядов графиков (колонки `AJ:AL`)

- `AJ` — имя ряда;
- `AK` — показывать ряд на относительном графике (`1/0`);
- `AL` — показывать ряд на абсолютном графике (`1/0`).

В шаблоне создаются чекбоксы, привязанные к `AK:AL`, поэтому можно включать/выключать ряды перед запуском без редактирования кода.

## 4. Алгоритм макроса

Для каждого включённого генератора (`J=1`):

1. **Определение точки старта**:
   - авто: последняя точка **внутри** `fнч` перед первым выходом частоты за `fнч` (отдельно в `Summary` фиксируется время первого выхода);
   - вручную: ближайшая точка к `U3`.

2. **Количественный критерий** на интервале `X5`:
   - `dF` = максимальное по модулю отклонение от `fном`;
   - `dFr` = отклонение за зоной нечувствительности;
   - `Pтреб_исх = -100/S · Pном/fном · Kд(t) · dFr`;
   - применяется **капинг по Pmax/Pmin**: `Pтреб = clip(Pтреб_исх, -max(0, P0-Pmin), max(0, Pmax-P0))`. Если произошло ограничение — статус становится `Ограничен Pmax/Pmin`;
   - `Pфакт = Pтек − P0`;
   - проверяется знак и уровень `|Pфакт|/|Pтреб|` с допуском `X6`.
   - Если `Колич. % > 100 + X6` — в `Summary` ставится метка `Перерегулирование`.
   - **Амплитуда** `= 100·|Pтреб_исх|/Pном` (по исходному, неограниченному значению — чтобы корректно характеризовать масштаб возмущения) и метка `Слабое / Норма / Избыточное`.

3. **Качественный критерий** (если `K=1`) — все цели **масштабируются к фактическому `Pтреб`** (а не к фиксированному % `Pном`), поэтому критерий одинаково работает и при контрольных испытаниях (ступень 10 % `Pном` → `Pтреб = Pном·dp10%/100`), и при мониторинге реальных событий любой амплитуды:
   - к моменту `t5` отработано не менее `dP5/dP10 · |Pтреб|` (по умолчанию 5/10 = 50 % `Pтреб`);
   - к моменту `t10` отработан полный `Pтреб` (с тем же знаком, возможны перерегулирования);
   - «установившееся» значение мощности (среднее за `X10` последних секунд качественного окна) сравнивается со **средним `Pтреб(t)` в том же окне** (с учетом `Kд(t)`), допуск `Уст_допуск % Pном` — только если `R=1`.
     - Если к концу окна частота восстановилась, `Pтреб_ср ≈ 0` — генератор тоже должен вернуться к `P0`.
     - Если частота всё ещё отклонена, генератор должен держать соответствующий `Pтреб`.

4. **Характер переходного процесса** по траектории `ΔP(t)` на `[Старт ; Старт + t10]`:
   - считаются значимые экстремумы (> 5 % амплитуды);
   - 0–1 экстремумов → `Монотонный`;
   - 2–3 экстремумов с убывающими амплитудами → `Апериодический`;
   - ≥ 4 экстремумов или не убывающие амплитуды → `Колебательный`.

5. **Листы на генератор**:
   - `<Ст>_<Ген>` — параметры + временной ряд + вспомогательные колонки для маркеров;
   - `<Ст>_<Ген>_Граф` — отдельный лист с двумя графиками:
     - верхний: относительные ряды (`dP`);
     - нижний: абсолютные ряды (`P, МВт`) + `Частота, Гц` на вторичной оси.
     В блоке **Вывод** дополнительно показывается фактический `Kд` и примененный профиль `Kд(t)`.

Для каждой станции, у которой хотя бы у одного генератора `Q=1`:

- строится общий суммарный лист `<Ст>_Сумма` (данные) + `<Ст>_Сумма_Граф` (график);
- дополнительно по каждому уникальному значению колонки `Паропровод` строятся подгрупповые листы:
  - `<Ст>_Сумма_<Паропровод>` + `<Ст>_Сумма_<Паропровод>_Граф`;
- в сумму включаются только генераторы с мгновенной мощностью выше `X7`;
- **`Pmax_сум` / `Pmin_сум`** считаются как сумма `Pmax` / `Pmin` включённых в работу генераторов. Располагаемый резерв и капинг `Pтреб_сум` работают как для отдельного генератора; на графике добавлены уровни `ΔPmax_сум` и `ΔPmin_сум`.

- колонки `P, T, U, V` — зелёный `ОК`, красный `Нарушение`;
- колонка `W Кач.уст` — зелёный `ОК`, оранжевый `Нарушение` (как предупреждение);
- колонка `P Колич. статус` — жёлтый при `Ограничен Pmax/Pmin`;
- колонка `AH Ограничение` — жёлтый при `Pmax` / `Pmin`;
- колонка `Q Перерегулирование` — жёлтый при `Да`;
- колонка `R Характер процесса` — зелёный `Монотонный`, жёлтый `Апериодический`, красный `Колебательный`;
- колонка `N Масштаб события` — синий `Слабое`, жёлтый `Избыточное`;

Справа (начиная со столбца `AK`) выводится версионный штамп: версия, время запуска, длительность, имя книги и число строк в `RawData`.

## 6. Лист `Log`

При запуске автоматически формируется лист `Log` с предупреждениями и служебными сообщениями:

- разрывы по времени в `RawData` > 5 с;
- значения `S`, `fнч`, `Kд` вне типичных диапазонов;
- `t10 < t5`;
- частично заполненная колонка `Паропровод` в рамках станции;
- `Pmax < Pmin`, `Pmax > 1.3·Pном`, `Pmin < 0`;
- резерв в требуемую сторону `< 5 % Pном` (возможное ограничение по диапазону);
- `INFO` о случаях капинга `Pтреб` по `Pmax` / `Pmin` с указанием исходного и применённого значений;
- другие замечания.

Если замечаний нет, в листе остаётся строка `INFO — Замечаний не обнаружено`.

## 7. Листы по генераторам и станциям

Листы по генераторам:
- `<Ст>_<Ген>` — параметры (включая `Pуст_сред`, `Характер`, `Экстремумов`, `Амплитуда`) и таблица временного ряда с вспомогательными колонками;
- `<Ст>_<Ген>_Граф` — график + блок **Вывод**.

Листы суммарные:
- `<Ст>_Сумма` + `<Ст>_Сумма_Граф`;
- `<Ст>_Сумма_<Паропровод>` + `<Ст>_Сумма_<Паропровод>_Граф` — если в `Config` заполнена колонка `Паропровод`.

## 8. Типовые ошибки и замечания

- `Не найдена колонка мощности/частоты ...` — проверьте `Config` (C/D) и заголовки в `RawData`.
- `Пропущен: не заполнены обязательные параметры` — не заданы `Pном`, `S`, `fнч` или `Kд`.
- `Нет первичного отклика; проверьте генератор/датчик` — в `Summary` появляется, если `P0 ≈ 0` и `Pфакт ≈ 0` (генератор не в работе или отсутствует сигнал).
- `Слабое возмущение (< 3 %Pном)` / `Избыточное (> 10 %Pном)` — дополнительные пометки в `Примечание`: событие за пределами типичного нормативного диапазона 3–10 % Pном.

## 9. Рекомендуемый порядок работы

1. Выгрузите все сигналы в `RawData` и заполните `Config`.
2. При необходимости нажмите **Применить пресеты типов** — макрос заполнит пустые нормативные параметры по типу оборудования.
3. Нажмите **Запустить мониторинг ОПРЧ**.
4. Проверьте `Log` — если есть `WARN`, уточните конфигурацию.
5. Проанализируйте `Summary`: по цветам легко видно проблемные генераторы.
6. По каждому проблемному генератору откройте `<Ст>_<Ген>_Граф` — блок **Вывод** содержит краткую сводку, график подсвечивает `t5`, `t10` и момент выхода за `fнч`.
7. Для повторного запуска с другими параметрами нажмите **Очистить результаты** и затем снова **Запустить мониторинг ОПРЧ**.
